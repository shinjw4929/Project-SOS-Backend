## 주석 검토 실행

$ARGUMENTS가 있으면 해당 내용을 검토 범위/대상으로 반영한다.

### 1단계: 변경된 파일 파악

다음을 병렬로 실행하여 검토 대상 파일을 결정한다:
- `git diff HEAD --name-only` (staged + unstaged 변경 파일 목록)
- `git status` (untracked 새 파일 확인, `-uall` 플래그 사용 금지)
- 사용자가 특정 파일을 지정한 경우 해당 파일만 대상

변경된 소스 파일(`.cpp`, `.h`, `.hpp`)만 검토 대상으로 삼는다. untracked 소스 파일도 포함한다. 문서, 설정, 빌드 산출물은 제외.

### 2단계: 파일별 주석 검토

각 파일에 대해 다음을 검사한다:

**Doxygen 주석 (`///`, `/** */`)**:
- 클래스/함수의 역할 설명이 실제 동작과 일치하는가
- 새로 추가된 기능이 문서화되었는가
- `@param`, `@return`, `@brief` 등이 정확한가

**인라인 주석 (`//`)**:
- 주석이 코드 동작을 정확히 설명하는가
- 변경된 로직에 대한 주석이 업데이트되었는가
- 더 이상 존재하지 않는 동작을 설명하는 주석이 남아있지 않은가

**블록 주석 (`/* */`)**:
- 알고리즘/설계 의도 설명이 현재 구현과 일치하는가

**설정 파일 주석 (`#`, `--`)**:
- TOML/YAML/SQL 파일의 주석이 실제 설정과 일치하는가

**삭제 대상 주석**:
- 실제 구현과 일치하지 않는 주석 → **삭제**
- 코드만으로 자명한 동작에 대한 주석 → **삭제**
- 삭제/변경된 로직을 설명하는 주석 → **삭제**

### 3단계: 결과 보고 및 수정

검토 결과를 표로 정리한 뒤, 즉시 수정을 실행한다. 별도 사용자 확인을 기다리지 않는다.

| 파일 | 위치 | 조치 | 이유 |
|------|------|------|------|
| 파일명 | 라인 번호 | 삭제 / 수정 | 문제 설명 |

**조치 원칙**:
- 구현과 불일치하는 주석은 수정이 아닌 **삭제**가 기본이다. 주석을 현재 구현에 맞게 고쳐 쓰는 것은 코드가 자명하지 않을 때만 한다.
- Doxygen 주석은 실제 동작에 맞게 **수정**한다.
- 보고와 수정을 한 단계에서 처리한다.
