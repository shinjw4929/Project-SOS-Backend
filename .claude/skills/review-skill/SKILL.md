## 역할

스킬 SKILL.md 파일을 검토하여 토큰 효율성, 기존 스킬과의 중복, 프로젝트 적합성을 점검하고 개선안을 제시한다.

$ARGUMENTS가 있으면 스킬명 또는 경로로 해석한다. 없으면 `.claude/skills/*/SKILL.md`를 전체 스캔하여 요약 리포트를 출력한다.

## 검토 절차

### 1단계: 대상 확정

1. $ARGUMENTS가 스킬명이면 `.claude/skills/[$ARGUMENTS]/SKILL.md`를 읽는다.
2. $ARGUMENTS가 경로이면 해당 파일을 읽는다.
3. $ARGUMENTS가 없으면 전체 스킬 스캔 모드로 전환한다 (3단계 참조).

### 2단계: 개별 스킬 검토

다음 4개 관점에서 순차 검토한다.

**A. 토큰 효율성**

| 점검 항목 | 기준 | 측정 방법 |
|---|---|---|
| 파일 크기 | 5KB 이하 권장, 6KB 이상 경고 | `wc -c` |
| 줄 수 | 200줄 이하 (hard limit) | `wc -l` |
| 대형 문서 참조 | 10KB 이상 문서를 직접 Read하는 절차가 있는지 | 본문에서 `docs/` 경로 추출 -> `wc -c`로 크기 확인 |
| auto-call 체인 깊이 | `/스킬명` 호출이 2단계 이상 중첩되는지 | 본문에서 `/스킬명` 패턴 추출 -> 해당 스킬의 auto-call 재귀 추적 |
| 불필요한 반복 | 같은 문서를 여러 단계에서 읽는 절차가 있는지 | 본문 내 Read 대상 경로 중복 확인 |

**B. 기존 스킬 중복**

1. `.claude/skills/SKILLS.md`에서 전체 스킬 목록을 읽는다.
2. 대상 스킬의 목적(description)과 기존 스킬의 목적을 비교한다.
3. 중복 판단 기준:
   - 동일 입력 -> 동일 출력을 생성하는 스킬이 이미 있는가
   - 기존 스킬에 옵션/인자를 추가하면 대체 가능한가
4. 중복률이 높으면 기존 스킬 확장을 제안한다.

**C. 프로젝트 적합성**

| 점검 항목 | 기준 |
|---|---|
| allowed-tools 최소화 | 코드 수정이 없으면 Edit/Write 제외, 사용자 확인이 없으면 AskUserQuestion 제외 |
| Tier 배치 | SKILLS.md의 계층 구조에서 적절한 위치에 있는가 |
| 프론트매터 형식 | name(kebab-case), description(한국어 1줄), allowed-tools 존재 |
| 한국어 작성 | 본문이 한국어로 작성되었는가 |
| 출력 형식 | 최종 출력 템플릿이 코드 블록으로 정의되어 있는가 |

**D. 구조 품질**

| 점검 항목 | 기준 |
|---|---|
| $ARGUMENTS 처리 | 첫 단계에서 인자 파싱 + fallback(질문 또는 기본값) 명시 |
| 단계별 절차 | 번호 매긴 단계가 있는가, 각 단계의 입출력이 명확한가 |
| 주의사항 섹션 | 존재하는가, 엣지 케이스를 다루는가 |
| 과잉 설계 | 하나의 명확한 목적만 수행하는가 |

### 3단계: 전체 스캔 모드 ($ARGUMENTS 없음)

$ARGUMENTS 없이 호출된 경우:

1. `.claude/skills/*/SKILL.md`를 Glob으로 수집한다.
2. 각 스킬에 대해 `wc -c`와 `wc -l`을 실행한다.
3. 2단계 A(토큰 효율성)만 전체 스킬에 적용하여 요약 테이블을 출력한다.
4. 경고 기준을 넘는 스킬만 상세 검토 대상으로 표시한다.

### 4단계: 결과 보고

**개별 스킬 검토 시:**

```
## 스킬 검토 결과: {스킬명}

### 토큰 효율성
| 항목 | 값 | 판정 |
|---|---|---|
| 파일 크기 | NKB | Pass / 경고 |
| 줄 수 | N줄 | Pass / 초과 |
| 대형 문서 참조 | [목록] | Pass / 경고 |
| auto-call 체인 | 깊이 N | Pass / 경고 |

### 기존 스킬 중복
| 유사 스킬 | 중복 영역 | 판정 |
|---|---|---|
| ... | ... | 독립 / 확장 권장 |

### 프로젝트 적합성
| 항목 | 판정 | 비고 |
|---|---|---|
| allowed-tools | Pass / 과잉 | ... |
| Tier 배치 | 적절 / 부적절 | ... |
| 프론트매터 | Pass / 수정 필요 | ... |

### 구조 품질
| 항목 | 판정 |
|---|---|
| $ARGUMENTS 처리 | Pass / 누락 |
| 단계별 절차 | Pass / 불명확 |
| 주의사항 | Pass / 누락 |

### 개선안
| # | 항목 | 현재 | 제안 | 이유 |
|---|---|---|---|---|
| 1 | ... | ... | ... | ... |

### 최종 판정
(적합 / 개선 필요 / 재설계 필요)
```

**전체 스캔 시:**

```
## 전체 스킬 토큰 효율성 스캔

| 스킬 | 크기 | 줄 수 | auto-call | 대형 참조 | 판정 |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | Pass / 경고 |

### 경고 대상
- {스킬명}: {사유}

### 최적화 제안
- ...
```

## 주의사항

- **읽기 전용**: 이 스킬은 코드나 스킬 파일을 수정하지 않는다. 개선안만 제시한다.
- **사실 기반**: 파일 크기, 줄 수, 참조 경로 등 측정 가능한 데이터로만 판단한다.
- **과잉 지적 금지**: 기준치 이내인 항목은 "Pass"로 처리하고 상세 설명을 붙이지 않는다.
