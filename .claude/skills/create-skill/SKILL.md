---
name: create-skill
description: 기존 스킬 패턴을 참조하여 새 스킬을 생성합니다. 프론트매터, 단계별 절차, 주의사항 구조를 자동으로 맞추고 200줄 이내로 작성합니다.
allowed-tools: Read, Write, Edit, Grep, Glob, Bash, AskUserQuestion
---

## 역할

기존 스킬의 구조와 컨벤션을 분석하여 새 스킬 SKILL.md를 생성하고, SKILLS.md 인덱스에 등록한다.

$ARGUMENTS가 있으면 스킬명/목적으로 반영한다. 없으면 사용자에게 질문한다.

## 생성 절차

### 1단계: 요구사항 파악

$ARGUMENTS에서 다음을 추출한다:

| 항목 | 설명 |
|---|---|
| **스킬명** | kebab-case (예: `review-perf`, `gen-test`) |
| **목적** | 이 스킬이 해결하는 문제 |
| **트리거 조건** | 사용자가 언제 이 스킬을 호출하는지 |
| **입출력** | $ARGUMENTS로 받는 것, 최종 출력 형식 |

추출할 수 없는 핵심 항목(스킬명, 목적)이 없으면 `AskUserQuestion`으로 질문한다.

### 2단계: 유사 스킬 탐색

1. `.claude/skills/SKILLS.md`를 읽어 기존 스킬 목록과 계층 구조를 파악한다.
2. 새 스킬과 **가장 유사한 기존 스킬 1~2개**를 선정하여 SKILL.md를 읽는다.

**유사도 판단 기준** (우선순위):
- 동일 Tier (오케스트레이션 / 후처리 / 검증 / 독립 도구)
- 유사한 입출력 구조 (코드 분석형, 코드 수정형, 보고형)
- 유사한 allowed-tools 구성

### 3단계: 스킬 설계

레퍼런스 스킬에서 다음 패턴을 추출하여 새 스킬에 적용한다:

**A. 프론트매터**
```yaml
---
name: {kebab-case}
description: {한국어 1줄, 무엇을 하는지 + 언제 사용하는지}
allowed-tools: {필요한 도구만. 코드 수정 없으면 Edit/Write 제외}
---
```

**B. 본문 구조** (레퍼런스와 동일한 골격 유지)
```markdown
## 역할
(1~2문장)

## {실행 절차 제목}

### 1단계: {입력/수집}
### 2단계: {분석/처리}
### N단계: {출력}

## 주의사항
```

**C. allowed-tools 선택 가이드**

| 용도 | 도구 |
|------|------|
| 코드/문서 읽기 | Read, Grep, Glob |
| 코드 수정 | Edit, Write |
| 셸 명령 (git, cmake) | Bash |
| 병렬 작업 위임 | Agent |
| 다른 스킬 호출 | Skill |
| 사용자 확인 | AskUserQuestion |

### 4단계: SKILL.md 작성

설계를 바탕으로 `.claude/skills/{스킬명}/SKILL.md`를 작성한다.

**작성 규칙:**

1. **200줄 이내**: 프론트매터 포함 전체 파일이 200줄을 넘지 않는다. 넘으면 절차를 압축하거나 체크리스트를 별도 파일(`docs/Checklists/`)로 분리한다.
2. **레퍼런스 구조 준수**: 2단계에서 선정한 레퍼런스의 마크다운 구조(헤딩 깊이, 테이블 형식, 코드 블록 사용)를 따른다.
3. **$ARGUMENTS 처리**: 첫 단계에서 `$ARGUMENTS` 파싱 방법을 명시한다. 없을 때의 fallback(질문 또는 기본값)도 포함한다.
4. **출력 형식 명시**: 최종 출력을 코드 블록 템플릿으로 정의한다.
5. **자동 호출 연계**: 후처리가 필요하면 `/sync-comments`, `/sync-docs` 등 자동 호출을 명시한다. 코드를 수정하지 않는 스킬은 후처리 불필요.
6. **서브에이전트 전략**: 병렬 처리가 필요한 경우, 분기 조건(예: "파일 3개 이상")과 Agent에게 전달할 정보를 명시한다.
7. **한국어**: 프론트매터 description과 본문 모두 한국어로 작성한다.

### 5단계: SKILLS.md 인덱스 업데이트

`.claude/skills/SKILLS.md`에 새 스킬을 등록한다:

1. **스킬 계층 구조**: 적절한 Tier에 추가한다.
2. **시나리오**: 새 스킬이 사용되는 시나리오를 추가하거나, 기존 시나리오의 체이닝에 포함한다.
3. **공유 참조 문서**: 체크리스트를 별도 파일로 분리한 경우 테이블에 추가한다.

### 6단계: 결과 보고

```
## 스킬 생성 완료

### 생성 파일
| 파일 | 줄 수 |
|------|-------|
| .claude/skills/{name}/SKILL.md | N줄 |

### 레퍼런스
| 스킬 | 참조 이유 |
|------|-----------|
| ... | ... |

### Tier 배치
{Tier N}: {역할 설명}

### 사용법
/{스킬명} [인자 설명]
```

## 주의사항

- **200줄 hard limit**: 생성된 SKILL.md가 200줄을 초과하면 작성을 중단하고 압축한다. 상세 체크리스트는 `docs/Checklists/{스킬명}-checklist.md`로 분리하고 스킬에서 Read로 참조한다.
- **기존 스킬 중복 금지**: 기존 스킬과 목적이 80% 이상 겹치면 생성 대신 기존 스킬 확장을 제안한다.
- **과잉 설계 금지**: 스킬은 하나의 명확한 목적만 수행한다. 여러 목적을 합치지 않는다.
- **레퍼런스 필수**: 기존 스킬을 최소 1개 읽지 않고 작성하지 않는다.
