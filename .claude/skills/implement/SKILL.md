## 구현 실행

### 1단계: 요구사항 파악

$ARGUMENTS에서 다음을 추출한다:
- **대상**: 무엇을 구현하는가
- **목표**: 어떤 동작을 해야 하는가
- **범위**: 어떤 모듈/파일에 영향을 주는가

### 2단계: 패턴 탐색

`docs/Checklists/pattern-search-guide.md`를 로드하여 구현 유형에 맞는 참조 패턴을 탐색한다.

**규칙**:
- 최소 1개, 최대 3개 참조 패턴을 찾는다
- 수정 대상 파일 내 기존 코드가 최우선 참조
- 같은 모듈 > 다른 모듈 순으로 탐색
- 패턴을 못 찾으면 CLAUDE.md Development Guidelines 기준으로 구현

### 3단계: docs 참조

`docs/Systems/` 폴더에서 관련 모듈 문서를 읽어 현재 시스템 구조를 파악한다.

### 4단계: 코드 구현

참조 패턴과 CLAUDE.md Development Guidelines를 따라 코드를 작성한다.

**원칙**:
- 기존 패턴의 구조/네이밍/에러 처리 방식을 따른다
- 요청된 기능만 구현한다. 추가 기능, 리팩토링, 불필요한 추상화를 하지 않는다
- CMakeLists.txt에 새 소스 파일 반영 필요 여부를 확인한다
- Protobuf 변경 시 proto/ 파일 수정 + 코드젠 재실행

### 5단계: 검증

- 변경 파일이 기존 패턴과 일관되는지 확인
- 컴파일 오류가 예상되는 부분이 있는지 점검

### 6단계: 후처리 (자동)

구현 완료 후 다음 스킬을 순차 호출한다:
1. `/review-comments` — 변경 파일의 주석 정합성 점검
2. `/update-docs` — 관련 docs 문서 업데이트

### 7단계: 결과 보고

사용자에게 다음을 보고한다:
- 참조한 패턴 (파일:메서드)
- 변경/생성한 파일 목록
- 빌드/테스트 권고 (`/build`, `/test`)
