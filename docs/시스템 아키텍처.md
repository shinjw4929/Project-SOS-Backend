## 서비스 구성

### Game Server (Unity)

| 항목 | 값 |
|------|------|
| 기술 | Unity Netcode for Entities |
| 포트 | 7979 |
| 역할 | 게임 로직, ECS 시뮬레이션 |
| 로그 | 로컬 파일 (`persistentDataPath/Logs/`) |
| 로그 형식 | `<timestamp> \| <LEVEL> \| [<S/C>:<Category>] <message>, key=value` |

Game Server는 Unity 프로젝트(Project-SOS)에서 실행되며, 이 저장소에는 포함되지 않는다.
백엔드 인프라는 Game Server의 로그 파일을 Vector로 수집하여 ClickHouse에 저장한다.

### Room Server (C++)

| 항목 | 값 |
|------|------|
| 기술 | C++20 / Boost.Asio / Protobuf |
| 포트 | 8080 (클라이언트), 8081 (내부) |
| 역할 | 방 생성/참가/시작 관리, 토큰 발급, 세션 관리 (1~8인 PvE) |
| 로그 | Docker stdout |
| 프로토콜 | TCP + Protobuf Envelope (4byte LE 프레이밍) |
| 상세 | [Room Server](Systems/Room%20Server.md) |

### Chat Server (C++)

| 항목 | 값 |
|------|------|
| 기술 | C++20 / Boost.Asio / Protobuf |
| 포트 | 8082 (클라이언트), 8083 (내부) |
| 역할 | 로비/전체/귓속말 채팅 |
| 로그 | Docker stdout |
| 프로토콜 | TCP + Protobuf ChatEnvelope (4byte LE 프레이밍) |
| 상세 | [Chat Server](Systems/Chat%20Server.md) |

### Redis

| 항목 | 값 |
|------|------|
| 이미지 | redis:7-alpine |
| 포트 | 6379 |
| 역할 | Room/Chat 공통 운영 저장소 |
| 인증 | `--requirepass` (환경변수 `REDIS_PASSWORD`) |
| 용도 | 방 관리, 토큰 TTL, Rate Limit, 채팅 히스토리 |

### Logging Pipeline

| 항목 | 값 |
|------|------|
| 수집 | Vector 0.41.1 |
| 저장 | ClickHouse 24.3 |
| 시각화 | Grafana 11.4.0 |
| 상세 | [로깅 파이프라인](로깅%20파이프라인.md) |

## 서비스 간 통신

### 클라이언트 접속 흐름

```
[클라이언트] ──RoomListRequest──► [룸 서버 :8080]
                                    │ 방 목록 응답
[클라이언트] ──CreateRoom/JoinRoom──► [룸 서버]
                                    │ 방 참가
                                    │ 전원 레디 → 방장 StartGame
                                    ├── GameStart(token, session_id) → 전체 클라이언트
                                    └── SessionCreated(session_id, players) → [채팅 서버 :8083]

[클라이언트] ──ChatAuth──► [채팅 서버 :8082]
                              │ 로비 채널 참가
                              │ 게임 시작 시 → 세션 채널 전환

[클라이언트] ──token──► [게임 서버 :7979]
                            │ TokenValidateRequest → [룸 서버 :8081] → Redis 검증
                            │ 검증 성공 → Hero 생성, 게임 시작
```

### 게임 종료 흐름

```
[게임 서버] ──SlotReleased──► [룸 서버 :8081]
                                    │ active_sessions 정리
                                    └── SessionEnded(session_id) → [채팅 서버 :8083]
                                                                        │ 세션 채널 삭제
                                                                        │ 플레이어 → 로비 복귀
```

### 비정상 종료 복구

```
게임 서버 크래시:
  하트비트 미수신 → game_server:{serverId} TTL 90초 만료
  → 해당 서버 세션 자동 정리 → 슬롯 복구

채팅 서버 재시작:
  Redis chat:session:{session_id}에서 세션 복구
  클라이언트 재접속 → ChatAuth 재전송 → 채널 복귀
```

## 데이터 저장소

### Redis 키 구조

| 키 패턴 | 타입 | TTL | 용도 | 서비스 |
|---------|------|-----|------|--------|
| `token:{uuid}` | STRING | 60s | 입장 토큰 (JSON: player_id, session_id) | Room |
| `active_sessions` | SET | - | 진행 중인 세션 | Room |
| `game_server:{serverId}` | STRING | 90s | 게임 서버 하트비트 | Room |
| `rate:{ip}` | STRING | 10s | Rate Limit 카운터 | Room |
| `chat:session:{session_id}` | HASH | 7200s | 세션 플레이어 매핑 (player_id → player_name) | Chat |
| `chat:player:{player_id}` | STRING | 7200s | 플레이어 → 세션 ID 매핑 | Chat |
| `chat:name:{player_id}` | STRING | 7200s | 플레이어 표시명 | Chat |
| `chat:rate:{player_id}` | STRING | 5s | 채팅 Rate Limit | Chat |
| `chat:history:{session_id}:ALL` | LIST | 7200s | 세션 채팅 히스토리 (최근 20개, Protobuf 직렬화) | Chat |

### ClickHouse 스키마

| 테이블 | 엔진 | 용도 |
|--------|------|------|
| `service_events` | MergeTree | 전 서비스 통합 이벤트 (90일 TTL) |
| `violation_rate_by_build` | SummingMergeTree | 빌드별 위반 시간당 집계 (MV) |
| `error_rate_by_service` | SummingMergeTree | 서비스별 에러 시간당 집계 (MV) |

## 로그 형식 (전 서비스 공통)

```
<timestamp> | <LEVEL> | [<World/Category>] <message>, key=value, ...
```

| 서비스 | 예시 |
|--------|------|
| Game Server | `2026-03-16 12:34:43.415 \| INFO    \| [S:Network] Client connected, networkId=1` |
| Room Server | `2026-03-16 10:00:01.100 \| INFO    \| [Room] Room started, room_id=abc, players=4` |
| Chat Server | `2026-03-16 10:01:01.100 \| INFO    \| [Chat] Message sent, channel=ALL` |

Game Server는 `[S:Category]` / `[C:Category]` (Server/Client), C++ 서버는 `[Category]`로 구분된다.

## 채팅 채널

| 채널 | 범위 | 히스토리 | 생명주기 |
|------|------|---------|---------|
| LOBBY | 전체 접속자 | 없음 | 서버 시작 시 생성, 상시 유지 |
| ALL | 같은 세션 전체 | Redis, 최근 20개, TTL 2시간 | SessionCreated → SessionEnded |
| WHISPER | 1:1 | 없음 | player_id 직접 전달, 채널 없음 |

## 코드 구조

```
src/
├── common/                   # STATIC 라이브러리 (Room + Chat 공유)
│   ├── protocol/             # Codec<T> 템플릿 (Envelope, ChatEnvelope)
│   │                           4byte LE 프레이밍 + Protobuf 직렬화
│   ├── redis/                # RedisClient (redis-plus-plus 래퍼)
│   │                           SessionStore (세션/토큰 TTL 관리)
│   ├── ratelimit/            # RateLimiter (Redis INCR + EXPIRE)
│   └── util/                 # Config (nlohmann/json), Logger (spdlog),
│                               UuidGenerator (UUID v4)
│
├── room/                    # Room Server 실행 파일
│   ├── server/               # RoomServer (TCP accept :8080)
│   │                           ClientSession (클라이언트 연결)
│   ├── room/                 # RoomManager (방 생성/참가/시작)
│   │                           Room (방 상태 관리, 1~8인)
│   └── internal/             # GameServerChannel (:8081, 토큰 검증 + SlotReleased)
│                               ChatServerChannel (채팅 서버 연동)
│
└── chat/                     # Chat Server 실행 파일
    ├── server/               # ChatServer (TCP accept :8082)
    │                           ChatSession (클라이언트 연결)
    ├── channel/              # ChannelManager, LobbyChannel, SessionChannel(ALL)
    └── internal/             # InternalChannel (:8083, 룸 서버 연결)
```
