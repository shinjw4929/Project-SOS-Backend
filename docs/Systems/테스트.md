## Catch2 유닛 테스트

`tests/` — sos_common 라이브러리의 핵심 컴포넌트 검증.

### 빌드 및 실행

```bash
# Developer Command Prompt for VS 2022
cmake --build build --target unit-tests
cd build && ctest --output-on-failure
```

| 파일 | 대상 | 검증 내용 |
|------|------|----------|

## Python E2E 테스트

`scripts/` — Room/Chat 서버의 엔드투엔드 시나리오 검증.

### 사전 요구사항

- Python 3.10+, `pip install protobuf`
- Redis 실행 (`docker compose up -d redis`)
- 대상 서버 실행

### 공유 유틸리티

`proto_client.py`:
- `ProtoClient` — 4byte LE 프레이밍 비동기 TCP 클라이언트
- `TestRunner` — 테스트 결과 집계 + check 헬퍼
- 헬퍼 함수: `room_create`, `room_join`, `room_leave`, `room_toggle_ready`, `room_start_game`, `room_list`, `chat_auth`, `chat_send`

### test_room.py (Room Server)

```bash
python scripts/test_room.py
```

13개 시나리오:

| 테스트 | 검증 |
|--------|------|
| 방 생성 + 목록 조회 | CreateRoom, RoomListRequest 정상 동작 |
| 방 입장 | JoinRoom + RoomUpdate 브로드캐스트 |
| 꽉 찬 방 입장 거부 | ROOM_FULL RejectResponse |
| 존재하지 않는 방 | ROOM_NOT_FOUND |
| 중복 플레이어 | DUPLICATE_PLAYER |
| 이미 방에 있음 | ALREADY_IN_ROOM |
| 레디 토글 (On) | ToggleReady → RoomUpdate 반영 |
| 레디 토글 (Off) | 재토글 → 미준비 상태 복귀 |
| 비호스트 시작 거부 | NOT_HOST |
| 미준비 시작 거부 | NOT_ALL_READY |
| 정상 게임 시작 (2인) | GameStart 전체 수신, auth_token + session_id 존재 |
| 솔로 방 시작 | 호스트 1인 방 즉시 시작 |
| 호스트 퇴장 시 방 파괴 | ROOM_CLOSED → 나머지 멤버에게 전송 |

### test_chat.py (Chat + Room 통합)

```bash
# Room + Chat 서버 모두 실행 상태 필요
python scripts/test_chat.py
```

검증 시나리오:
- 로비 채팅 (ChatAuth → CHANNEL_LOBBY 브로드캐스트)

- Rate Limit 발동 검증
- 세션 전환 (로비 → 세션 → 로비 복귀)

## 부하 테스트

`scripts/test_load.py` — Python asyncio 기반 동시 접속 벤치마크.

### 실행

```bash
python scripts/test_load.py                    # 전체 시나리오 (기본 10명)
python scripts/test_load.py connect 50         # 동시 접속 50명
python scripts/test_load.py room_join 20       # 방 동시 입장 20명
python scripts/test_load.py chat 30            # 채팅 폭주 30명
python scripts/test_load.py lifecycle 10       # 전체 생명주기 10명
```

### 시나리오

| 시나리오 | 설명 | 측정 |
|---------|------|------|
| connect | N명 동시 접속 + 방 생성 | 총 소요시간, 클라이언트당 평균 |
| room_join | 1개 방에 N명 동시 입장 (최대 7명) | 입장 성공률, 소요시간 |
| chat | N명 로비 접속 후 5메시지씩 동시 전송 | 전송/수신 건수, Rate Limit 영향 |
| lifecycle | N/2쌍의 2인 방: 생성 → 입장 → 레디 → 시작 → 채팅 | E2E 성공률, 소요시간 |

출력: 시나리오별 대상/성공/실패/총시간/평균시간 요약 테이블.

## Vector 파싱 테스트

`infra/vector/vector-tests.toml` — Vector의 VRL 파싱 로직 검증 (10개 테스트).

### 실행

```bash
docker compose cp infra/vector/vector-tests.toml vector:/tmp/vector-tests.toml
MSYS_NO_PATHCONV=1 docker exec $(docker compose ps -q vector) \
  vector test \
  --config-toml /etc/vector/vector.toml \
  --config-toml /tmp/vector-tests.toml
```

| 테스트 | 주입 | 검증 |
