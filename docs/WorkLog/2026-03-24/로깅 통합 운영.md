# 로깅 통합 + 운영

> Phase 5 (Step 5-1 ~ 5-5). Docker 컨테이너화, 로그 파이프라인 통합, Grafana 대시보드/알림 구축.

---

## 개요

Phase 1~4에서 구현한 Room/Chat 서버를 Docker 이미지로 빌드하여 Compose 스택에 통합했다. Vector 정규식 버그를 수정하고, 서버 설정 기본값 템플릿을 생성하고, Grafana에 서버 전용 대시보드와 알림 규칙을 추가했다.

**핵심 수정**: Vector `parse_cpp` 정규식이 `\w+`로 카테고리를 매칭하여, `Room:Internal`, `Room:Chat`, `Chat:Internal` 등 콜론 포함 카테고리 28개 로그가 드롭되는 버그를 수정했다.

## Step 5-1: Vector 정규식 수정

### 문제

`infra/vector/vector.toml` line 68의 `parse_cpp` 정규식:

```
# 변경 전 (콜론 포함 카테고리 매칭 실패)
\[(?P<category>\w+)\]

# 변경 후
\[(?P<category>[\w:]+)\]
```

### 영향받는 카테고리

| 카테고리 | 사용처 | 로그 수 |
|----------|--------|---------|
| `Room:Internal` | GameServerChannel, GameServerSession | 12 |
| `Room:Chat` | ChatServerChannel | 6 |
| `Chat:Internal` | InternalChannel, InternalSession | 7 |

`Room`, `Chat`, `SessionStore` 등 콜론 없는 카테고리는 기존에도 정상 동작.

## Step 5-2: Docker 컨테이너화

### Dockerfile (프로젝트 루트)

멀티 스테이지 멀티 타겟 구조:

```
build-base (ubuntu:24.04)
  ├── 빌드 도구: GCC 14, CMake, Ninja, vcpkg
  ├── vcpkg.json COPY → vcpkg install (캐시 레이어 분리)
  └── 소스 COPY → cmake configure + build

room-server (ubuntu:24.04 minimal)
  └── build-base에서 room-server 바이너리 복사

chat-server (ubuntu:24.04 minimal)
  └── build-base에서 chat-server 바이너리 복사
```

vcpkg.json을 소스보다 먼저 COPY하여 의존성 레이어를 캐시한다. 소스 변경 시 vcpkg 설치(10~20분)를 건너뛸 수 있다.

### Config.cpp 환경변수 fallback

JSON 파일 로딩 후 환경변수로 오버라이드:

```cpp
if (auto* val = std::getenv("REDIS_HOST")) data_["redis_host"] = val;
if (auto* val = std::getenv("REDIS_PORT")) data_["redis_port"] = std::stoi(val);
if (auto* val = std::getenv("REDIS_PASSWORD")) data_["redis_password"] = val;
if (auto* val = std::getenv("CHAT_SERVER_HOST")) data_["chat_server_host"] = val;
if (auto* val = std::getenv("CHAT_SERVER_PORT")) data_["chat_server_port"] = std::stoi(val);
```

Docker 환경에서 Redis는 `redis` (Docker DNS), Chat Server는 `chat-server`로 접근해야 하므로 환경변수 오버라이드가 필수.

### docker-compose.yml 변경

주석 처리된 서버 서비스(line 68-80)를 실제 서비스로 교체:

| 서비스 | 빌드 타겟 | 포트 | 의존성 |
|--------|----------|------|--------|
| chat-server | `Dockerfile` target: chat-server | 8082, 8083 | redis (healthy) |
| room-server | `Dockerfile` target: room-server | 8080, 8081 | redis (healthy), chat-server (started) |

Room Server가 Chat Server에 SessionCreated/Ended를 전송하므로, `depends_on`에 `chat-server`를 추가했다.

### CMakeLists.txt 조건부 빌드

`.dockerignore`에서 `tests/`를 제외하므로 Docker 빌드 시 디렉토리가 없다. CMake configure 실패를 방지:

```cmake
if(EXISTS "${CMAKE_SOURCE_DIR}/tests")
    enable_testing()
    add_subdirectory(tests)
endif()
```

### .dockerignore

빌드 컨텍스트에서 불필요한 디렉토리 제외: `build/`, `.git/`, `docs/`, `infra/`, `tests/`, `scripts/`, `vcpkg_installed/`

## Step 5-3: Grafana 대시보드

기존 대시보드(`service-overview.json`, `game-events.json`) 패턴을 따라 2개 추가.

### Room Metrics (8패널)

| 패널 | 타입 | 쿼리 필터 |
|------|------|----------|
| Room Log Volume | timeseries (stacked) | `service='room'`, level별 분당 집계 |
| Room Created / Removed | timeseries | `Room created` / `room removed` 메시지 매칭 |
| Player Join / Leave / Disconnect | timeseries | 플레이어 입퇴장 이벤트 |
| Game Started | timeseries | `Game started` 분당 수 |
| Token / Session Events | timeseries | 토큰 검증/만료, 슬롯 해제 |
| Rate Limited Requests | timeseries | rate limit 관련 로그 |
| Internal Channel Log (Game/Chat Server) | table | `category IN ('Room:Internal', 'Room:Chat')` 최근 50개 |
| Room Server Errors | table | `level IN ('WARNING', 'ERROR')` 최근 50개 |

### Chat Metrics (9패널)

| 패널 | 타입 | 쿼리 필터 |
|------|------|----------|
| Chat Log Volume | timeseries (stacked) | `service='chat'`, level별 분당 집계 |
| Player Connect / Disconnect | timeseries | 접속/이탈 추이 |
| Authentication Events | timeseries | `Player authenticated (lobby/session)` 인증 추이 |
| Session Created / Ended | timeseries | 세션 생성/종료 추이 |
| Messages by Channel | timeseries | LOBBY/ALL/WHISPER 채널별 메시지 수 (`chat_messages` 테이블) |
| Rate Limited / Redis Errors | timeseries | Rate Limit + Redis 에러 |
| Chat Message Log | table | `chat_messages` 테이블 최근 50개 |
| Internal Channel Log (Room Server) | table | `category='Chat:Internal'` 최근 50개 |
| Chat Server Errors | table | `level IN ('WARNING', 'ERROR')` 최근 50개 |

데이터소스: `clickhouse-ds`, 테이블: `project_sos.service_events`

## Step 5-5: Grafana 알림 규칙

파일 기반 프로비저닝 (`infra/grafana/provisioning/alerting/`).

### 알림 규칙 (5개)

| 알림 | 조건 | 대기 시간 | 심각도 |
|------|------|----------|--------|
| Error Rate Spike | 5분간 ERROR 10건+ | 5분 | critical |
| Room Creation Failure | 5분간 Room ERROR 5건+ | 3분 | warning |
| Redis Connection Error | 5분간 Redis 에러 3건+ | 2분 | critical |
| Chat Server Disconnected | 5분간 Room:Chat 경고 3건+ | 3분 | warning |
| Internal Channel Error | 5분간 Internal 에러 3건+ | 3분 | warning |

### 프로비저닝 구조

```
infra/grafana/provisioning/alerting/
├── alert-rules.yml       # 알림 규칙 5개 (ClickHouse SQL 기반)
├── contact-points.yml    # 기본 알림 수신 (Grafana 내장)
└── policies.yml          # 라우팅 정책 (alertname + severity 그룹)
```

docker-compose.yml Grafana 볼륨에 마운트:
```yaml
- ./infra/grafana/provisioning/alerting:/etc/grafana/provisioning/alerting
```

개발 단계에서는 Grafana UI 알림만 사용하고, 추후 Slack/Discord webhook 추가 가능.

## 설계 결정

### 단일 Dockerfile 멀티 타겟

Room/Chat이 `src/common/`, `proto/` 등을 공유하므로 빌드 컨텍스트가 프로젝트 루트여야 한다. 단일 `build-base` 스테이지에서 빌드하고, 런타임 이미지는 서버별로 분리.

### 환경변수 fallback 방식

JSON 설정 파일을 기본으로 하되, 환경변수가 존재하면 오버라이드한다. Docker Compose에서 `REDIS_HOST: redis`, `CHAT_SERVER_HOST: chat-server` 등으로 서비스 간 Docker DNS를 주입한다.

### Room Server → Chat Server 의존성

`depends_on: chat-server: { condition: service_started }`를 사용하되, ChatServerChannel은 자체 재연결 메커니즘(5초 간격)이 있으므로 Chat Server 기동 지연에도 내성이 있다.
