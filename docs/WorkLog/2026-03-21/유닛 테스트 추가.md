# 유닛 테스트 추가

> Phase 1-2 완료 후, 외부 의존(Redis, TCP) 없이 단독 테스트 가능한 4개 컴포넌트에 대한 유닛 테스트 추가.

---

## 개요

Phase 6까지 테스트를 미루면 회귀 버그 발견이 늦어지므로, Catch2 프레임워크를 도입하여 Phase 1(common 라이브러리)과 Phase 2(Room Server 핵심)의 순수 로직 컴포넌트를 검증하는 유닛 테스트를 작성했다.

최종 결과: **32개 테스트 케이스, 144개 어서션, 전체 통과.**

---

## 파일 구조

```
tests/
├── CMakeLists.txt           # Catch2 find_package + unit-tests 타겟
├── test_Codec.cpp           # Codec<Envelope> 인코딩/디코딩 테스트
├── test_UuidGenerator.cpp   # UUID v4 생성기 테스트
├── test_Config.cpp          # JSON 설정 파싱 테스트
└── test_Room.cpp            # Room 로직 테스트
```

## 테스트 프레임워크

### Catch2 v3

vcpkg를 통해 Catch2 v3을 설치하고, `Catch2::Catch2WithMain`을 링크하여 별도 main 함수 없이 테스트를 실행한다.

| 패키지 | CMake 타겟 | 용도 |
|--------|-----------|------|
| catch2 | `Catch2::Catch2WithMain` | 테스트 프레임워크 + 자동 main |

### CMake 구성

단일 실행 파일 `unit-tests`에 4개 테스트 파일을 포함한다. `add_test()`로 CTest에 등록.

```cmake
find_package(Catch2 3 CONFIG REQUIRED)

add_executable(unit-tests
    test_Codec.cpp
    test_UuidGenerator.cpp
    test_Config.cpp
    test_Room.cpp
    "${CMAKE_SOURCE_DIR}/src/room/room/Room.cpp"
)

target_include_directories(unit-tests PRIVATE
    "${CMAKE_SOURCE_DIR}/src/room"
)

target_link_libraries(unit-tests PRIVATE
    Catch2::Catch2WithMain
    sos_common
)

include(CTest)
add_test(NAME unit-tests COMMAND unit-tests)
```

Room.cpp는 room-server 타겟에만 포함되어 있으므로 unit-tests 타겟에 직접 소스로 추가한다.

## 테스트 대상 및 케이스

### test_Codec.cpp (6 케이스)

`Codec<Envelope>` 템플릿 클래스의 length-prefixed 프레이밍(4바이트 LE + payload)을 검증한다.

| 케이스 | 검증 내용 |
|--------|----------|
| encode-decode roundtrip | 인코딩 후 디코딩하면 원본과 동일 |
| incomplete header | 4바이트 미만 데이터에서 nullopt 반환 |
| incomplete body | 헤더만 있고 바디 부족 시 nullopt, 나머지 도착 시 디코딩 성공 |
| 1MB+ rejection | MAX_MESSAGE_SIZE 초과 메시지 거부 (exception) |
| consecutive messages | 연속된 두 메시지를 순서대로 디코딩 |
| invalid protobuf bytes | 유효하지 않은 바이트에서 exception |

### test_UuidGenerator.cpp (4 케이스)

`sos::generateUuid()` 함수의 UUID v4 RFC 4122 준수를 검증한다.

| 케이스 | 검증 내용 |
|--------|----------|
| format | 36자, 하이픈 위치(8,13,18,23), 나머지 16진수 |
| version 4 bit | 인덱스 14 = '4' |
| variant bit | 인덱스 19 = 8/9/a/b |
| uniqueness | 1000회 생성 시 전체 유일 |

### test_Config.cpp (5 케이스)

`sos::Config` 클래스의 JSON 파싱과 기본값 폴백을 검증한다. `TempConfigFile` RAII 헬퍼로 임시 JSON 파일을 생성/삭제한다.

| 케이스 | 검증 내용 |
|--------|----------|
| JSON parsing | room_port, redis_host, max_rooms 파싱 |
| default values | 14개 accessor 전체 기본값 확인 |
| generic get\<T\> | 존재하지 않는 키에 대한 fallback 값 |
| nonexistent file | 파일 미존재 시 runtime_error |
| accessor override | 12개 accessor에 커스텀 값 설정 후 확인 |

### test_Room.cpp (17 케이스)

`sos::Room` 클래스의 방 관리 로직과 Protobuf 변환을 검증한다.

| 그룹 | 케이스 | 검증 내용 |
|------|--------|----------|
| 생성 | constructor | 호스트 자동 추가, 초기 상태 ROOM_WAITING |
| addPlayer | 성공/중복/만석/게임중 | 참가 성공, 중복 거부, 만석 거부, IN_GAME 거부 |
| removePlayer | 존재/미존재 | 퇴장 성공, 미존재 플레이어 false 반환 |
| toggleReady | 호스트/일반 | 호스트 토글 무시, 일반 플레이어 토글 동작 |
| canStart | 솔로/미준비/전원준비 | 호스트만 true, 미준비 false, 전원준비 true |
| isFull | 용량 반영 | 정원 도달 시 true |
| 상태 전이 | setState | WAITING → IN_GAME |
| playerIds | 전체 ID | 3명 추가 후 ID 목록 확인 |
| Protobuf | toRoomInfo/toRoomSummary | 필드 매핑 정확성, 플레이어 정보 포함 |

## 빌드 환경 이슈 및 해결

### vcpkg applocal.ps1 경로 버그

프로젝트 경로에 공백이 포함되어 있어 vcpkg의 DLL 복사 스크립트(`applocal.ps1`)가 `cmd.exe`의 중첩 따옴표 처리 버그로 실패한다.

**해결**: CMake configure 시 `-DVCPKG_APPLOCAL_DEPS=OFF` 캐시 변수를 전달하여 DLL 복사를 비활성화.

```bash
cmake --preset=default -DVCPKG_APPLOCAL_DEPS=OFF
```

### catch_discover_tests() 미사용

계획에서는 `catch_discover_tests()`를 사용하려 했으나, 이 역시 applocal 경로 버그를 트리거한다. `add_test(NAME unit-tests COMMAND unit-tests)`로 대체하여 단일 CTest 항목으로 등록한다. Catch2 내부에서 32개 케이스를 모두 실행한다.

### VS DevShell 필수

CLI에서 빌드할 때는 Developer Command Prompt 또는 VS DevShell 환경이 필요하다. 일반 bash 셸에서는 `INCLUDE`, `LIB` 환경변수가 없어 표준 헤더를 찾지 못한다.

cd build && ctest --output-on-failure
