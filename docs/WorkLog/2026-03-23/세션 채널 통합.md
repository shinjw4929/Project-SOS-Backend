# 세션 채널 통합

> Phase 4 Step 4-2. 내부 TCP 채널 + 세션 채널 생명주기 + 메시지 히스토리.

---

## 개요

Room Server가 게임 시작/종료 시 Chat Server에 `SessionCreated`/`SessionEnded`를 전달하여 세션 채널(ALL)을 자동 생성/삭제한다. Chat Server의 내부 포트 :8083에서 Room Server 연결을 수신한다.

---

## InternalChannel (:8083)

`127.0.0.1:8083`에 바인딩하는 localhost 전용 TCP acceptor. Room Server의 GameServerChannel(:8081)과 동일한 패턴.

```
InternalChannel(io_context, port, channel_manager)
  └── doAccept() → InternalSession 생성
```

---

## InternalSession

Room Server 연결을 관리한다. `Codec<ChatEnvelope>` 기반 프레이밍.

| 수신 메시지 | 처리 |
|------------|------|
| `SessionCreated` | ChannelManager::handleSessionCreated() |
| `SessionEnded` | ChannelManager::handleSessionEnded() |

하트비트/Rate Limit 없음 (내부 신뢰 연결).

## 세션 채널 생명주기

### SessionCreated 처리

```
SessionCreated(session_id, [{player_id, player_name}, ...]) 수신
  1. SessionChannel 생성 (expected_players 등록)
  2. Redis에 플레이어 정보 저장:
     - HSET chat:session:{session_id} {player_id} {player_name}
     - SETEX chat:name:{player_id} {player_name} 7200
     - EXPIRE chat:session:{session_id} 7200
```

### 클라이언트 세션 인증 (ChatAuth with session_id)

```
ChatAuth(player_id, session_id, player_name) 수신
  1. session_channels_에서 session_id 존재 확인 (없으면 실패)
  2. 세션 채널 members에 player_id 추가
  3. Redis에 매핑 저장:
     - HSET chat:session:{session_id} {player_id} {player_name}
     - SETEX chat:player:{player_id} {session_id} 7200
     - SETEX chat:name:{player_id} {player_name} 7200
  4. 히스토리 전달 (최근 20개)
```

### SessionEnded 처리

```
SessionEnded(session_id) 수신
  1. 세션 채널의 모든 멤버를 로비로 이동:
     - session_id 클리어
     - in_lobby = true
     - lobby_members_에 추가
  2. session_channels_에서 채널 삭제
  3. Redis 키 정리:
     - DEL chat:session:{session_id}
     - DEL chat:history:{session_id}:ALL
```

## 메시지 히스토리

세션 채널(ALL) 메시지를 Redis LIST에 저장하여 재접속 시 맥락을 제공한다.

### 저장

```
ChatSend(CHANNEL_ALL) 처리 시:
  1. ChatReceive를 Protobuf 직렬화
  2. LPUSH chat:history:{session_id}:ALL {serialized}
  3. LTRIM chat:history:{session_id}:ALL 0 19  (최대 20개 유지)
  4. EXPIRE chat:history:{session_id}:ALL 7200
```

### 조회 (세션 채널 참가 시)

```
sendHistory(player_id, session_id):
  1. LRANGE chat:history:{session_id}:ALL 0 19
  2. 역순으로 순회 (오래된 메시지부터)
  3. 각 항목을 ChatReceive로 역직렬화
  4. ChatEnvelope로 감싸서 플레이어에게 전송
```

---

## Redis 키 구조

| 키 | 타입 | TTL | 용도 |
|-----|------|-----|------|
| `chat:session:{session_id}` | HASH | 7200초 | 세션 플레이어 매핑 (field=player_id, value=player_name) |
| `chat:player:{player_id}` | STRING | 7200초 | 플레이어 → 세션 ID 역매핑 |
| `chat:name:{player_id}` | STRING | 7200초 | 플레이어 표시명 (재시작 시 복구용) |
| `chat:history:{session_id}:ALL` | LIST | 7200초 | 직렬화된 ChatReceive (최신이 앞, 최대 20개) |
