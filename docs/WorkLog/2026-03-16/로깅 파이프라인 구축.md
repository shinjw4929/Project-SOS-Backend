# 로깅 파이프라인 구축 (2026-03-16)

## 개요

Unity Game Server의 로컬 로그 파일을 Vector로 수집하여 ClickHouse에 적재하고, Grafana 대시보드로 시각화하는 파이프라인을 구축했다.

```
[Unity Game Server]
    GameLogger → com.unity.logging → 로컬 파일 (.log)
                                          ↓ tail
                                     [Vector Agent]
                                     파싱 + 필드 추출
                                          ↓
                                     [ClickHouse]
                                     service_events 테이블
                                          ↓
                                     [Grafana]
                                     대시보드 2개
```

## ClickHouse

### 테이블: `project_sos.service_events`

전 서비스(Game/Queue/Chat) 통합 테이블. 서비스별 확장 컬럼 포함.

| 컬럼 | 타입 | 용도 |
|------|------|------|
| timestamp | DateTime64(3, 'UTC') | 이벤트 시각 |
| service | LowCardinality(String) | game / queue / chat |
| level | LowCardinality(String) | DEBUG / INFO / WARNING / ERROR |
| category | LowCardinality(String) | Combat / Wave / Economy / Network / Movement / System |
| world | LowCardinality(String) | S (Server) / C (Client) |
| message | String | 로그 메시지 본문 |
| network_id | Int32 | 유저 식별 (Game) |
| ghost_id | Int32 | 엔티티 식별 (Game) |
| queue_depth | Int32 | 큐 깊이 (Queue) |
| match_wait_ms | Int32 | 매칭 대기시간 (Queue) |
| channel | LowCardinality(String) | 채팅 채널 (Chat) |

**파티셔닝**: `(service, toYYYYMMDD(timestamp))`
**정렬**: `(service, category, timestamp)`
**TTL**: 90일 자동 삭제

### Materialized Views

- `violation_rate_by_build_mv`: 빌드별 위반 유형 시간당 집계
- `error_rate_by_service_mv`: 서비스별 에러 시간당 집계

### ClickHouse 서버 설정 (`users.xml`)

| 설정 | 값 | 이유 |
|------|---|------|
| `input_format_skip_unknown_fields` | 1 | Vector가 보내는 메타 필드 무시 |
| `date_time_input_format` | best_effort | ISO 8601 (`2026-03-16T12:34:43.357Z`) 파싱 허용 |

## Vector

### 파이프라인 흐름

```
[Source: game_logs]     → file tail (/logs/game/**/*.log)
[Source: docker_logs]   → Docker stdout (vector.service 라벨)
         ↓
[Transform: parse_game] → 정규식 파싱 + 구조화 필드 추출
[Transform: parse_cpp]  → C++ 서버 로그 파싱 (Queue/Chat)
         ↓
[Transform: clean_fields] → Vector 메타 필드 제거 (file, host, source_type 등)
         ↓
[Sink: clickhouse]      → ClickHouse HTTP API (JSONEachRow)
```

### Game Server 로그 파싱

입력:
```
2026-03-16 12:34:43.357 | DEBUG | [S:Wave] Wave0 spawned, count=5
```

출력 (JSON):
```json
{
  "timestamp": "2026-03-16T12:34:43.357Z",
  "service": "game",
  "level": "DEBUG",
  "world": "S",
  "category": "Wave",
  "message": "Wave0 spawned, count=5",
  "network_id": 0,
  "ghost_id": 0
}
```

### 구현 중 해결한 문제

| 문제 | 원인 | 해결 |
|------|------|------|
| Vector가 기본 설정 파일 로드 | `vector.yaml`을 찾는데 `.toml`로 마운트 | `command: ["--config", "/etc/vector/vector.toml"]` 추가 |
| VRL `strip_whitespace` 에러 | `.message` 타입이 `any`이므로 fallible | `strip_whitespace!(.message)` (infallible 연산자) |
| VRL `upcase!` 에러 | 이미 infallible 입력에 `!` 불필요 | `upcase(strip_whitespace!(...))` |
| VRL `to_string!` 에러 | `to_string`은 infallible 함수 | `to_string(ch.v)` |
| ClickHouse 400 Bad Request | Vector 메타 필드가 테이블에 없음 | `clean_fields` transform에서 `del(.file)` 등 제거 |
| ClickHouse DateTime 파싱 실패 | ISO 8601 `T...Z` 포맷 미지원 | `users.xml`에 `date_time_input_format: best_effort` |
| ClickHouse TTL 에러 | `DateTime64`를 TTL에 직접 사용 불가 | `TTL toDate(timestamp) + INTERVAL 90 DAY DELETE` |
| 로그 메시지 끝에 `\r` | Windows 줄바꿈 (`\r\n`) | `strip_whitespace!(.message)` |

## Grafana

### 접속 정보

- URL: `http://localhost:3000`
- 기본 계정: `admin` / `admin`
- 데이터소스: ClickHouse (uid: `clickhouse-ds`, 프로비저닝 자동 생성)

### 대시보드

#### Service Overview

| 패널 | 타입 | 내용 |
|------|------|------|
| Log Volume (per minute) | timeseries | 분당 로그 수, 레벨별 스택 |
| Level Distribution | piechart | DEBUG/INFO/WARNING/ERROR 비율 |
| Category Distribution | piechart | Combat/Wave/Economy/Network 비율 |
| Events by Category (per minute) | timeseries | 카테고리별 분당 이벤트 수 |
| Warnings & Errors | timeseries | WARNING/ERROR만 시계열 |
| Total Events / Warnings / Errors | stat | 기간 내 총 건수 |
| Recent Warnings & Errors | table | 최근 Warning/Error 100건 목록 |

#### Game Events

| 패널 | 타입 | 내용 |
|------|------|------|
| Wave Transitions | table | Wave 전환 이벤트 목록 |
| Kill Count (per minute) | timeseries | 분당 킬 수 추이 |
| Economy Events | timeseries | Build/Production 이벤트 추이 |
| Network Events | table | 접속/연결 이벤트 목록 |
| Enemy Spawns (per minute) | timeseries | 분당 적 스폰 수 |
| Movement Warnings | timeseries | Enemy Stuck, Path Failures 등 |
| Hero Deaths & Game Over | table | Hero 사망 + GameOver 이벤트 목록 |

### 대시보드 쿼리 규칙

- 시계열 패널: `format: 1`, `ORDER BY time ASC`
- 테이블 패널: `format: 2`, timestamp를 `toString(timestamp) AS time`으로 변환
- 시간 범위: `$__fromTime` / `$__toTime` 매크로 사용

## 운영 가이드

### 시작

```bash
cd Project-SOS-Backend
docker compose up -d
```

### 중지

```bash
docker compose down
```

### ClickHouse 초기화 (데이터 삭제 포함)

```bash
docker compose down
docker volume rm project-sos-backend_clickhouse_data
docker compose up -d
```

### Vector 체크포인트 초기화 (로그 재수집)

```bash
docker exec project-sos-backend-vector-1 bash -c "rm -rf /var/lib/vector/*"
docker compose restart vector
```

### 로그 확인

```bash
docker logs project-sos-backend-vector-1      # Vector
docker logs project-sos-backend-clickhouse-1   # ClickHouse
docker logs project-sos-backend-grafana-1      # Grafana
```

### ClickHouse 직접 쿼리

```bash
docker exec project-sos-backend-clickhouse-1 bash -c "clickhouse-client --password dev123 --query 'SELECT count() FROM project_sos.service_events'"
```

또는 브라우저에서 `http://localhost:8123/play` (user: `default`, password: `dev123`)
