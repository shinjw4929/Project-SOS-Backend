# 로깅 통합 + 운영 — 통합 체크리스트

## Context

Phase 1~4 완료로 Room Server, Chat Server가 모두 구현된 상태. 서버를 Docker 컨테이너로 패키징하고, 로그 파이프라인(Vector -> ClickHouse -> Grafana)에 통합하며, 운영 대시보드와 알림을 추가한다.

**핵심 발견**: Vector `parse_cpp` 정규식이 `\w+`로 카테고리를 매칭하여, `Room:Internal`, `Room:Chat`, `Chat:Internal` 등 콜론 포함 카테고리 27개 로그가 드롭되는 버그 존재.

---

## 현재 상태 분석

### spdlog 로그 형식

`src/common/util/Logger.cpp`에서 커스텀 포매터 구현 완료:

```
패턴: %Y-%m-%d %H:%M:%S.%e | LEVEL   | [Category] message
출력: 2026-03-18 10:00:01.100 | INFO    | [Room] Room created, room_id=abc
```

Vector `parse_cpp` 정규식과 기본적으로 일치하나, 콜론 포함 카테고리(`Room:Internal` 등) 매칭 실패.

### 로그 카테고리 현황

| 카테고리 | 사용처 | 로그 수 | Vector 매칭 |
|----------|--------|---------|------------|
| `Room` | RoomServer, ClientSession, RoomManager | 다수 | 정상 |
| `Room:Internal` | GameServerChannel, GameServerSession | 14 | **실패** |
| `Room:Chat` | ChatServerChannel | 6 | **실패** |
| `Chat` | ChatServer, ChatSession, ChannelManager | 다수 | 정상 |
| `Chat:Internal` | InternalChannel, InternalSession | 7 | **실패** |
| `SessionStore` | SessionStore | 다수 | 정상 |

### Docker 현황

- Dockerfile: 미생성
- docker-compose.yml: 서버 서비스 주석 처리 상태 (line 68-80)
- 빌드: MSVC/Windows 전용, Linux 빌드 미검증
- Config: JSON 파일 로딩 구현됨, 환경변수 오버라이드 미지원

### Grafana 대시보드 현황

| 대시보드 | 패널 | 대상 |
|---------|------|------|
| service-overview.json | 9 | 전체 서비스 (로그 볼륨, 레벨/카테고리 분포) |
| game-events.json | 7 | Game Server (Wave, 킬, 경제, 스폰) |
| **Room Metrics** | - | **미구현** |
| **Chat Metrics** | - | **미구현** |

### Grafana 알림 현황

- 알림 규칙: 미구현
- 프로비저닝: alerting 디렉토리 미생성

**목표**: 콜론 포함 카테고리(`Room:Internal`, `Room:Chat`, `Chat:Internal`)가 Vector에서 정상 파싱되도록 수정

- [ ] `infra/vector/vector.toml`: parse_cpp 정규식 `\w+` -> `[\w:]+`

> 상세: [01-로그 형식 적용.md](01-로그%20형식%20적용.md)

### Step 5-4: 설정 파일

**목표**: 서버 설정 기본값 템플릿 생성 (Graceful Shutdown은 이미 완료)

- [ ] `config/server_config.json`: 전체 설정 기본값 템플릿 생성 (Room 14개 + Chat 10개, 총 24개)

### Step 5-2: Docker 컨테이너화

**목표**: Room/Chat 서버를 Docker 이미지로 빌드하고 Compose 스택에 통합

- [ ] `Dockerfile` (루트): 멀티 스테이지 멀티 타겟 (build-base -> room-server / chat-server)
- [ ] `.dockerignore`: 빌드 컨텍스트 최적화
- [ ] `docker-compose.yml`: 서버 서비스 활성화 (context/target 구조)
- [ ] `src/common/util/Config.cpp`: 환경변수 fallback (REDIS_HOST, REDIS_PORT, REDIS_PASSWORD, CHAT_SERVER_HOST, CHAT_SERVER_PORT)
- [ ] `CMakeLists.txt` (루트): Docker 빌드 시 tests/ 미포함 대응 (`if(EXISTS)` 가드)
- [ ] `.env.example`: REDIS_HOST 등 추가

> 상세: [03-Docker 컨테이너화.md](03-Docker%20컨테이너화.md)

## 위험 요소

| 위험 | 영향 | 대응 |
|------|------|------|
| Linux 빌드 실패 | Docker 이미지 빌드 불가 | Dockerfile에 충분한 빌드 의존성 설치 (libssl-dev 등) |
| Docker 빌드 시간 | 첫 빌드 10-20분 | BuildKit 캐시: vcpkg.json을 소스보다 먼저 COPY |
| ClickHouse 쿼리 증가 | 대시보드 4개 + 알림 5개 = ~37 쿼리/30초 | 초당 1.2 쿼리, ClickHouse 부하 무시 가능 |

## 검증 방법

1. `docker compose up --build` 로 전체 스택 기동
2. Room/Chat 서버 컨테이너 stdout에서 로그 형식 확인
3. ClickHouse에서 `Room:Internal` 카테고리 로그가 정상 저장되는지 확인
4. Grafana (localhost:3000)에서 Room Metrics / Chat Metrics 대시보드 확인
5. 알림 규칙이 Grafana Alerting 탭에 표시되는지 확인
