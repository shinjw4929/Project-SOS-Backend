# Step 5-1: Vector 정규식 수정

## Context

spdlog 커스텀 포매터(`Logger.cpp`)는 이미 Vector 파싱 형식과 일치한다. 그러나 Vector `parse_cpp` 정규식이 `\w+`로 카테고리를 매칭하여, 콜론 포함 카테고리 27개 로그가 `abort`로 드롭된다.

---

## 영향 분석

### 현재 정규식

```
r'^(?P<ts>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\.\d{3}) \| (?P<level>\w+)\s+\| \[(?P<category>\w+)\] (?P<msg>.*)'
```

`\w+` = `[a-zA-Z0-9_]` — 콜론(`:`) 미포함.

### 드롭되는 로그 (27개)

| 카테고리 | 파일 | 로그 수 | 포함 레벨 |
|----------|------|---------|----------|
| `Room:Internal` | GameServerChannel.cpp, GameServerSession.cpp | 14 | info, warn, error, debug |
| `Room:Chat` | ChatServerChannel.cpp | 6 | info, warn |
| `Chat:Internal` | InternalChannel.cpp, InternalSession.cpp | 7 | info, warn, error |

내부 채널 관련 **모든** 로그(에러 포함)가 ClickHouse에 저장되지 않는다.

---

## 수정

### [ ] `infra/vector/vector.toml` (line 68)

```diff
- r'^(?P<ts>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\.\d{3}) \| (?P<level>\w+)\s+\| \[(?P<category>\w+)\] (?P<msg>.*)'
+ r'^(?P<ts>\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}\.\d{3}) \| (?P<level>\w+)\s+\| \[(?P<category>[\w:]+)\] (?P<msg>.*)'
```

`\w+` -> `[\w:]+` 변경으로 `Room:Internal`, `Room:Chat`, `Chat:Internal` 매칭.

---

## 검증

- Vector 재시작 후 `Room:Internal` 카테고리 로그가 ClickHouse `service_events` 테이블에 저장되는지 확인
- 기존 카테고리(`Room`, `Chat`, `SessionStore`)가 여전히 정상 파싱되는지 확인
