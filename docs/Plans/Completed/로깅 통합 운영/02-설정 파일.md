# Step 5-4: 설정 파일

## Context

Config 클래스(`src/common/util/Config.h/cpp`)는 JSON 파일 로딩을 지원하며, 두 서버 모두 `config/server_config.json` 경로에서 설정을 읽도록 구현되어 있다. 파일이 없으면 기본값으로 동작한다.

Graceful Shutdown은 이미 완료 — 두 서버 모두 `boost::asio::signal_set(io_context, SIGINT, SIGTERM)` 구현됨.

---

## 수정

### [ ] `config/server_config.json` 생성

```json
{
  "room_port": 8080,
  "internal_port": 8081,
  "redis_host": "127.0.0.1",
  "redis_port": 6379,
  "redis_password": "",
  "max_rooms": 100,
  "max_players_per_room": 8,
  "token_ttl_seconds": 60,
  "heartbeat_timeout_seconds": 30,
  "game_server_heartbeat_ttl_seconds": 90,
  "rate_limit_max": 20,
  "rate_limit_window_seconds": 10,
  "game_server_host": "127.0.0.1",
  "game_server_port": 7979,
  "chat_server_host": "127.0.0.1",
  "chat_server_port": 8083,
  "chat_port": 8082,
  "chat_internal_port": 8083,
  "chat_heartbeat_timeout_seconds": 90,
  "chat_rate_limit_max": 10,
  "chat_rate_limit_window_seconds": 5,
  "chat_max_message_length": 200,
  "chat_history_size": 20,
  "chat_session_ttl_seconds": 7200
}
```

Room Server는 `room_*` / `internal_*` / `redis_*` / `game_server_*` / `chat_server_*` 필드를 읽고, Chat Server는 `chat_*` / `redis_*` 필드를 읽는다. 각 서버가 불필요한 필드를 무시하므로 단일 파일로 관리 가능.

---

## 검증

- Room Server / Chat Server가 `config/server_config.json` 정상 로드 확인
- 파일 없이 실행 시 기본값으로 정상 동작 확인
