# Project-SOS 로깅 시스템 구축 계획

## Context

Project-SOS는 Game Server(Unity), Queue Server(C++), Chat Server(C++) 3개 서비스로 구성된다. 모든 서비스의 로그를 하나의 파이프라인으로 수집하여 ClickHouse에 저장하고, Grafana로 시각화/알림한다.

### 서비스 맵

| 서비스 | 기술 스택 | 포트 | 운영 데이터 | 로그 수집 방식 |
|--------|----------|------|-----------|--------------|
| Game Server | Unity Netcode for Entities | 7979 | ECS 인메모리 | 로컬 파일 → Vector tail |
| Queue Server | C++ / Boost.Asio | 8080, 8081 | Redis | stdout → Vector Docker log driver |
| Chat Server | C++ / Boost.Asio | 8082, 8083 | Redis | stdout → Vector Docker log driver |

### 왜 ClickHouse 단독인가 (MongoDB 제외)

1. **Vector에 MongoDB sink가 없음**: 계획 당시 `type = "mongodb"`로 작성했으나 Vector는 네이티브 MongoDB sink를 제공하지 않는다. 커스텀 HTTP 어댑터가 필요해져 파이프라인이 복잡해진다.
2. **로그 파싱으로는 MongoDB 스키마를 못 채움**: MongoDB 계획의 nested document(entity 상태, command_trail 배열)는 Tier 1의 단순 텍스트 로그에서 복원 불가.
3. **ClickHouse가 포렌식 조회도 커버**: `WHERE ghost_id = 42 AND session_id = 'abc'` 같은 개별 레코드 조회가 ms 단위로 가능.
4. **3개 서비스 × 2개 DB = 관리 비용 과다**: Redis + ClickHouse 2개로 충분.
5. **필요 시 추가**: 영구 채팅 히스토리, 유저 프로필 등 OLTP 영속 데이터가 필요해지면 그때 MongoDB 또는 PostgreSQL을 추가한다.

## 전체 아키텍처

```
[Game Server (Unity)]──→ 로컬 파일 ──→ [Vector Agent] ──→ [ClickHouse] ──→ [Grafana]
[Queue Server (C++)] ──→ Docker stdout ──┘                    │            대시보드
[Chat Server (C++)] ──→ Docker stdout ───┘                    │            알림
                                                              │
[Redis] ← Queue/Chat 공통 운영 저장소 (세션, 토큰, 캐시)        │
                                                         90일 TTL 자동 삭제
```

## Tier 1: Unity 내부 로컬 파일 로깅 (구현 완료)

> 상세 내용은 [Docs/로깅 시스템.md](../../로깅%20시스템.md) 참조

### 구현 상태

- **패키지**: `com.unity.logging: 1.3.10`
- **초기화**: `GameBootStrap.Initialize()` (World 생성 전 1회)
- **파일 경로**: `persistentDataPath/Logs/{yyyy-MM-dd}/game_{HHmmss}.log`
- **로테이션**: 10MB × 5 롤링 (세션당 50MB 상한)
- **보존**: 30일 초과 폴더 자동 삭제
- **래퍼**: `GameLogger` (Burst 호환, `[MethodImpl(AggressiveInlining)]`)
- **로그 포인트**: 11개 시스템에 적용 완료

### 로그 출력 형식

```
2026-03-15 14:40:45.030 | INFO    | [C:Network] Connected to server
2026-03-15 14:41:06.445 | INFO    | [S:Economy] Build succeeded, networkId=1, gridX=175, gridY=120
2026-03-15 14:42:37.757 | WARNING | [S:Movement] Enemy stuck, idx=42, pos=(15,23), dest=(30,10), cause=partial
```

## Tier 2: ClickHouse + Grafana — 통합 로그 저장소

> 상세 내용은 [로깅 Tier2 ClickHouse Grafana.md](로깅%20Tier2%20ClickHouse%20Grafana.md) 참조

### 핵심 요약

- **용도**: 포렌식 조회 + 집계/추이 분석 + 실시간 대시보드 + 알림 (단일 DB로 통합)
- **수집 대상**: Game Server(파일), Queue Server(Docker stdout), Chat Server(Docker stdout)
- **수집**: Vector Agent (네이티브 ClickHouse sink)
- **스키마**: `service_events` 테이블 (서비스 공통 컬럼 + 서비스별 확장 컬럼)
- **대시보드**: Service Overview, Game Violations, Queue Metrics, Chat Metrics, Error Dashboard
- **알림**: 빌드 회귀, WallPenetration 급증, 에러 급증, 큐 대기시간 이상
- **보존**: 90일 TTL 자동 삭제

## 인프라 구성

### Docker Compose

```yaml
# docker-compose.yml
services:
  # === 공통 인프라 ===
  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
    volumes:
      - redis_data:/data

  clickhouse:
    image: clickhouse/clickhouse-server:24.3
    ports:
      - "8123:8123"   # HTTP
      - "9000:9000"   # Native
    volumes:
      - clickhouse_data:/var/lib/clickhouse
      - ./infra/clickhouse/init.sql:/docker-entrypoint-initdb.d/init.sql
    environment:
      CLICKHOUSE_DB: project_sos
      CLICKHOUSE_USER: default
      CLICKHOUSE_PASSWORD: ${CH_PASSWORD:-dev123}

  vector:
    image: timberio/vector:0.41.1-debian
    volumes:
      - ./infra/vector/vector.toml:/etc/vector/vector.toml
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ${GAME_LOG_PATH}:/logs/game:ro
    depends_on: [clickhouse]
    environment:
      BUILD_VERSION: ${BUILD_VERSION:-dev}
      SESSION_ID: ${SESSION_ID:-local}
      SERVER_ID: ${SERVER_ID:-dev-1}
      CH_PASSWORD: ${CH_PASSWORD:-dev123}

  grafana:
    image: grafana/grafana:11.4.0
    ports: ["3000:3000"]
    volumes:
      - grafana_data:/var/lib/grafana
      - ./infra/grafana/datasources.yml:/etc/grafana/provisioning/datasources/ds.yml
    environment:
      GF_INSTALL_PLUGINS: grafana-clickhouse-datasource
    depends_on: [clickhouse]

  # === 게임 서비스 ===
  queue-server:
    build: ./src/queue
    ports: ["8080:8080", "8081:8081"]
    depends_on: [redis]
    labels:
      vector.service: "queue"

  chat-server:
    build: ./src/chat
    ports: ["8082:8082", "8083:8083"]
    depends_on: [redis]
    labels:
      vector.service: "chat"

volumes:
  redis_data:
  clickhouse_data:
  grafana_data:
```

## 구현 순서

### Phase 1: Tier 1 (구현 완료)
Unity 로컬 파일 로깅. `com.unity.logging` + `GameLogger` 래퍼.

### Phase 2: Tier 2 인프라 구축 (구현 완료)

> 구현 상세: [로깅 파이프라인](../../로깅%20파이프라인.md)

| Step | 작업 | 산출물 |
|------|------|--------|
| 1 | `infra/` 디렉토리 구조 생성 | 폴더 구조 |
| 2 | ClickHouse 초기화 SQL 작성 | `infra/clickhouse/init.sql` |
| 3 | Vector 설정 작성 (Game Server 파일 수집) | `infra/vector/vector.toml` |
| 4 | Grafana 데이터소스 프로비저닝 | `infra/grafana/datasources.yml` |
| 5 | Docker Compose 작성 | `docker-compose.yml` |
| 6 | `docker compose up` → 파이프라인 동작 검증 | 로그 적재 확인 |

### Phase 3: Grafana 대시보드 (구현 완료)

| Step | 작업 |
|------|------|
| 7 | Service Overview 대시보드 구성 (9패널) |
| 8 | Game Events 대시보드 구성 (7패널) |
| 9 | 알림 규칙 추가 (빌드 회귀, 에러 급증) — 미구현 |

### Phase 4: C++ 서버 로그 통합

| Step | 작업 |
|------|------|
| 10 | C++ 서버 로그 출력 형식 정의 (stdout, 구조화 텍스트) |
| 11 | Vector 설정에 Docker log source 추가 |
| 12 | Queue/Chat 서비스 대시보드 추가 |

## 검증 방법

1. **Tier 1**: Editor PlayMode → `persistentDataPath/Logs/` 로그 파일 생성 + 내용 확인
2. **Tier 2 파이프라인**: `docker compose up` → ClickHouse SQL 클라이언트에서 `SELECT * FROM service_events LIMIT 10` 확인
3. **Grafana**: `localhost:3000` 접속 → ClickHouse 데이터소스 연결 → 대시보드 패널에 데이터 표시 확인
4. **알림**: 테스트 로그 삽입 → 알림 규칙 트리거 확인
